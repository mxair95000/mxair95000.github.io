Fortuna 網路投保平台：網頁介面設計

Design Breif｜設計概述

富達線上保險平台是個能尋找和購買最適合個人保險方案的工具。只要輸入簡單的個人資料和實名驗證，就能快速從不同的保險公司中，找到最適合自己的保險產品，更棒的是還能直接線上購買。從比較到結帳都能在線上完成，免除了找業務的麻煩！



Design Process｜設計過程

在此專案中，主要負責介面設計、原型製作；並參與定義需求的過程。在定義需求階段透過競品分析和參考使用者研究的資料，確認出產品市場定位，和團隊討論確認產品之 User Story、功能規格。在介面設計階段透過 UI Flow、線框圖、Mockup，最後製作出動態原型以及一份 UI Guideline.




Product Positioning｜產品定位

Market Analysis｜市場現況分析

保險產品根據投保的標的，可以分為「產物險」「人壽險」兩種。若以服務方式來看，以販售自家產品為目標的稱為「保險公司」，若是根據消費者需求，代理申請不同家公司產品的則為保險經紀人公司。目前大多數能線上投保的公司以販售產物險的保險公司為主。人壽險的線上網站以查閱、媒合預約業務員臨櫃投保為主。Fortuna 富達定位為能夠一站式尋找、購買適合保險產品的線上保險經紀人服務，可購買的產品包含人壽險和產物險兩類產品。



Persona + 5W1H｜定義使用者

現代年輕人剛畢業踏入職場，對保險和理財等的知識較無概念，也不太喜歡接觸保險業務員；資訊取得方式上搜尋能力強，也常使用搜尋引擎查找相關資訊。Fortuna 瞄準這樣的需求，透過線上比較不同公司之保險產品、並直接購買保險的方式，讓年輕人可以靠自己在網路上找到最適合自己的保險方案。



Prototype Iteration｜原型迭代

在介面迭代的兩個月中，介面從 Functional Map 發展至 UI Flow 與 Wireframe。建立以對話式的介面流程，一步一步帶領使用者探索適合自己的保險方案，並加以購買。



Final Design：富達一站式保險平台

查找與比較保險方案

使用者進入 Fortuna 首頁後，可以依據自己想保險的種類進行查找配對，如意外險、汽機車險、平安險等。封面文案也會根據時間、登入與否進行變化。保險產品搜尋的會以卡片的方式摘要呈現，使用者看到希望可以比較的保險商品，可以點選卡片上「加入比較」即可以暫存該產品，點選「呈現比較」按鈕，則可以將暫存的產品一次攤開，比較之間的異同。底部進度條簡明的呈現使用者在購買保險過程中的進展程度。



線上購買保險方案

若已經登入帳號，並進行實名制驗證後。點選產品上的「立即購買」，介面會根據保險種類的不同而更動要輸入的資訊。如旅平險需要旅遊地區、意外險需確認健康狀態等。等此幫助服務判斷最適合的保險種類。確認完各項保險條款，和合約，並綁定信用卡後即可以付款購買。



帳號申請與實名驗證

使用者點選登入或在點選購買時未登入，都會開啟帳號申請/登入頁，為使初次使用、尚未打算在此平台購買使用者也能將 Foruna 作為產品搜尋/比較工具，因此申請會員僅使用電話號碼即可註冊，減少所需填寫資料。

點開「我的帳號」中有該使用者的會員資料，除基本資料外，也有投保中的保險方案與合約資料和業務員聯絡方式

UI Guideline｜介面標準文件

為了使網頁前端與其他人員能夠更有效率的開發網站。在設計 UI 之外，也整理一份 UI Guideline ，以及使用相關工具和風格的內部文件，幫助產品製作上更為順利，也讓他人交接工作時能夠更快的進入狀況。
